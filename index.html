ARCHIVO DE TERMINOLOGÍA

Ingeniería de Software / Programación / Estándares de Calidad

API

Desarrollo

Application Programming Interface. Conjunto de reglas y especificaciones que las aplicaciones pueden seguir para comunicarse entre sí. Sirve como un contrato entre distintos sistemas de software.

AGREGADO

Arquitectura / DDD

En Domain-Driven Design, conjunto de objetos relacionados que se tratan como una unidad para garantizar consistencia de invariantes y responsabilidades del dominio.

AUTENTICACIÓN

Seguridad

Proceso que verifica la identidad de un usuario o sistema (por ejemplo, contraseña, token o certificado) para permitir acceso a recursos protegidos.

AUTORIZACIÓN

Seguridad

Determinación de los permisos que tiene un usuario autenticado para realizar acciones o acceder a recursos específicos dentro de un sistema.

BUFFER

Sistemas

Área de memoria temporal usada para almacenar datos mientras se transfieren entre dos entidades con velocidades diferentes, mejorando eficiencia y rendimiento.

BROKER

Mensajería

Componente intermedio que recibe, enruta y entrega mensajes entre productores y consumidores, desacoplando sistemas y soportando modelos pub/sub y cola.

CONCURRENCIA

Programación

Capacidad de un sistema para ejecutar múltiples tareas que avanzan de forma overlapping en el tiempo; incluye conceptos como hilos, asincronía y sincronización.

CI/CD

DevOps

Prácticas de Integración Continua y Entrega/Despliegue Continuo que automatizan la construcción, pruebas y despliegue de software de forma segura y repetible.

CONTENEDOR

Infraestructura

Tecnología que empaqueta una aplicación y sus dependencias en una unidad aislada y ligera, facilitando la portabilidad entre entornos.

DOMINIO

Modelado

Conjunto de conceptos, reglas y entidades que representan el problema que la aplicación intenta resolver; base para diseñar modelos y lógica de negocio.

DESPLIEGUE

Operaciones

Acción de entregar y configurar una aplicación en un entorno (producción, staging) para que esté disponible a usuarios o sistemas.

ENCRIPTACIÓN

Seguridad

Proceso de transformar datos legibles en una forma cifrada para proteger su confidencialidad; requiere claves para desencriptar y recuperar la información original.

ENDPOINT

APIs

URL o dirección de un recurso expuesto por un servicio que permite realizar operaciones (GET, POST, PUT, DELETE) sobre datos o funcionalidades.

FALLBACK

Resiliencia

Mecanismo alternativo que se activa cuando la opción principal falla, por ejemplo usar datos en caché, servir una respuesta degradada o repetir con política distinta.

FORK

Control de versiones

Copia de un repositorio que crea un nuevo punto de inicio para desarrollar de forma independiente y proponer cambios mediante pull requests.

FRAMEWORK

Herramientas

Conjunto estructurado de bibliotecas, convenciones y herramientas que facilita el desarrollo de aplicaciones reduciendo trabajo repetitivo.
